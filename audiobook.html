项目概述

这是一个泛化的有声小说自动生成系统。输入一部小说文本，系统自动识别对话角色、匹配声线、生成多角色配音音频，输出完整的有声小说。已完成小说《语冰人》43 章全流程验证。

处理流程

📖 章节解析
→
💬 对话识别
→
👤 角色提取
→
🎭 声线匹配
→
🔊 音频生成
→
📻 章节合成

关键技术点

  • 动态角色提取：基于 LLM 自动识别对话归属角色，无需预定义角色列表
  • 声线映射引擎：覆盖 581 个角色声线（6 款游戏），自动匹配最优参考音频
  • 数据清洗策略：短句过滤（<10字→旁白）、非对话内容识别、置信度评分
  • 分段 AI 管道：规则引擎处理 80%+ 简单情况，AI 辅助处理 20% 复杂场景
  • 双格式输出：Markdown 电子书（阅读）+ GPT-SoVITS 格式（批量音频生成）

实战数据

  • 处理规模：43 章节，3248 句原始对话 → 1554 句有效对话
  • 角色数量：10 个主要角色 + 动态配角
  • 高置信度对话占比：64.2%
  • 声线覆盖率：6 款游戏的 581 个角色参考音频

技术栈

  • TTS 引擎：GPT-SoVITS v2 — 少样本声音克隆，支持中文
  • AI 辅助：DeepSeek API — 角色识别与声线匹配
  • 后端：Python FastAPI — 任务管理、状态追踪、音频流
  • 输出格式：Markdown 电子书 + 分章节 WAV/MP3 音频

迭代经验

本项目经历了多次重构，核心教训：

  • ❌ 预定义角色列表 → ✅ 动态角色提取（泛化）
  • ❌ 大 chunk LLM 处理 → ✅ 小 chunk + 规则引擎优先
  • ❌ 直接生成音频 → ✅ 先验证数据质量，再批量生成
  • ❌ 纯 AI 驱动 → ✅ 规则引擎 80% + AI 20% 的分层策略
